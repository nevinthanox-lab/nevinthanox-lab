## 🚀 Featured Projects

### 🔹 [n8n Enterprise AI Ecosystem](https://github.com/nevinthanox-lab/n8n-enterprise-ai-ecosystem)
Three n8n agents running end-to-end, all reporting into one shared error-alert pipeline:

- **Lead Qualification Agent** — researches every inbound lead live via SerpAPI before GPT-4o-mini scores it, so the score is grounded in real evidence instead of a guess. Hot leads route straight to Slack/Gmail; every result is logged to Airtable.
- **RAG-Powered Support Agent** — answers customer questions using only internal docs. Anything it doesn't know, it hands off to a human on Slack instead of guessing.
- **Ticket Triage Orchestrator** — auto-resolves routine IT tickets and pauses high-risk ones for human approval. Hardened against database injection attacks.

### 🔹 [Agentic MLOps Production Pipelines](https://github.com/nevinthanox-lab/agentic-mlops-production-pipelines)
Classical ML models handing decisions off to guardrailed agents:

- **ChurnGuard** — an XGBoost model predicts churn risk and SHAP explains why. A LangGraph agent drafts a retention offer for each at-risk account, but it only goes out after a human approves it on Slack.
- **PromptShield** — a fine-tuned DistilBERT model flags adversarial prompts. A 4-agent CrewAI pipeline triages, sanitizes, judges, and escalates them — running fully offline on local Ollama.
- **SentinelOps** — a 3-model anomaly ensemble flags unusual telemetry. A RAG pipeline suggests the right fix from a runbook library, escalating to a human whenever it isn't confident.

## 🛡️ Engineering Principles

| Principle | What it means in practice |
|:---|:---|
| **Fail-closed by default** | Every agent errs toward escalation — never a silent failure |
| **Schema-validated outputs** | Strict JSON contracts, zero hallucinated structure |
| **Human-in-the-loop** | High-risk decisions always wait for explicit approval |
| **Classical ML → Agent handoff** | Models score and explain; agents execute the decision |
